# Regression tests

HTTP integration tests for `meetmanager-app`. No SPLASH involvement — the suite
exercises the FastAPI backend end to end and validates the generated Lenex
export structurally.

## Prerequisites

- Docker + `docker compose`
- Python 3.10+
- Ports `8000` (backend) and `8001` (frontend) free

## Run

```bash
pip install -r tests/requirements-test.txt
pytest tests/ -v
```

The session fixture wipes and rebuilds the docker stack (`docker compose down -v && up --build -d`) — **any data already in your local `pgdata` volume will be destroyed**. Set `MEETMGR_KEEP_STACK=1` to skip the final teardown.

## Fixtures

- `tests/fixtures/meet_template.lxf` — the meet structure (Gatineau 2026, 57 events spanning all junior + adult age groups, no Masters).
- `tests/fixtures/test_entries.lxf` — generated by `tests/generate_test_entries.py`. Deterministic (seed `20260509`): 5 clubs × 5 categories × 2 genders × 2 athletes = 100 athletes, with birth years tuned to land in each category given an AGEDATE of Dec 31 2026.
- `tests/fixtures/test_results.lxf` — generated by `tests/generate_test_results.py` from the meet template + entries fixture. Each athlete gets ~3 RESULTs on age/gender-appropriate events with plausible swim times.

Regenerate manually if you change the generators:

```bash
python tests/generate_test_entries.py --out tests/fixtures/test_entries.lxf
python tests/generate_test_results.py \
    --meet tests/fixtures/meet_template.lxf \
    --entries tests/fixtures/test_entries.lxf \
    --out tests/fixtures/test_results.lxf
```

## Coverage

| Area | Tests |
|---|---|
| Setup / smoke | meet upload event count, entries upload counts, status, meet-info |
| Auth | admin PIN, club PIN, invalid PIN |
| Registration view | suggested_age_code per birth-year, adult-only categories, junior categories visible, gender filter sanity |
| Registration write | create + delete round-trip, category re-register flow |
| Export | valid `.lxf` zip, ENTRY count = registration count, every ENTRY has `eventid` + `agegroupid`, `eventid` resolves to a defined EVENT, `agegroupid` resolves to a defined AGEGROUP on that event |
| Results | upload returns `times_updated`, `athletes_skipped == 0`, `/status` reflects new best_times count, an athlete's registration response surfaces `best_time_scm_ms` |
